## Objective

In this lab an external LED had to be connected to the free GPIO pin on the STM32 Nucleo-F401RE board through a current-limiting resistor and configured that pin as a push-pull output in CubeMX. Firmware that repeatedly toggles the pin inside the main poop with a visible delay between state changes should be written as well. Then using oscilloscope, the behavior should be observed on waves.

What the GPIO port and pin arguments represent and how HAL function maps to setting and clearing bits in the GPIO output register should also be explained.

## Code explanation

Since most of the project code has been generated by CubeMX, 
![image](images/Pasted%20image%2020260320183528.png)

the actual code we write is tiny and directly aimed at blinking LED and is located inside the main while loop 

```cpp
while (1)
{
    HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_6);
    HAL_Delay(500);
}
```

The first thing inside the `main(void)` function is `HAL_initi` and is used for HAL startup. It can be compared to the basic initialization part of Arduino that happens before `setup()`, where the core library prepares the hardware and system functions. After that, `SystemClock_Config()` configures the clock setup, and `MX_GPIO_Init()` is used for GPIO initialization.

`HAL` stands for Hardware Abstraction Layer and can be understood as the Arduino's digitalWrite() which is basically a set of libraries doing most of the work for us. `GPIOA` stands for General Purpose Input Output port A. `TogglePin` does exactly what is says. 

So, it will turn on/off LED keep the state for 500 milliseconds, then will turn it off.
